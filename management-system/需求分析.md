# 需求分析

1. 实体
   + 用户：用户名、密码、手机号、性别、年龄、学院、年级、社团职位、用户状态、注册时间
   + 社团：社团名称、社团介绍、成立时间
   + 社团公告：公告标题、公告内容、阅读人数、发布时间、是否置顶
   + 社团活动：活动名称、活动内容、参与人数、发布时间、
   + 值班表：值班时间、值班学生
   + 收支：收支类型、收支名称、收支金额、备注、余额
2. 实体间关系
   - 一个社团有多个用户
   - 一个用户可以加入多个社团
   - 一个社团有多个公告
   - 一个社团有多个社团活动
   - 一个社团有一个值班表
   - 一个社团有一个收支情况
   - 一个用户可以参加多个社团活动
   - 一个社团活动有多个用户参加
